# 御方通和门店采购与合伙人收益 V1

## 目标

本模块在既有总部供应链与门店库存 V1 上增加采购收益快照和合伙人可见台账，不建立第二套商品、物流、库存、订单或支付系统。

## 采购流程

1. 总部在既有 `yfth_supply_catalog` 配置 CRMEB 商品、SKU、采购价、起订量和包装倍数。
2. 只有当前门店的店长可以创建采购单。
3. 创建采购单时冻结商品实付金额、规则版本和五级合伙人链路。
4. 总部沿用既有采购审核、发货流程。
5. 门店沿用既有收货入库流程；首次有效入库确认采购收益。
6. 重复发货、重复收货和重复收益确认由事务锁和唯一索引拦截。

采购收益基数为采购商品实际金额，不含运费。商品折扣应先进入采购单成交快照，再以快照金额计算收益。

## 五级采购收益

初始规则如下，总部可以发布新规则版本，比例允许为 0：

| 职级 | 初始比例 |
| --- | ---: |
| 县级合伙人 | 20% |
| 地级合伙人 | 10% |
| 省级合伙人 | 5% |
| 大区总监 | 3% |
| 平台董事 | 1% |

历史采购单始终读取自己的冻结快照，发布新比例不会改写历史收益。

## 层级与就近分配

系统按以下顺序确定采购单的直接县级合伙人：

1. 门店已有有效合伙人绑定；
2. 门店已有有效开店来源；
3. 门店所在区县的有效县级合伙人；
4. 同城市有效县级合伙人；
5. 同省有效县级合伙人；
6. 其他有效县级合伙人。

同一层级内依次按服务区域优先级、有效开店负载、任职时间和 UID 稳定排序。已有明确来源时不会被就近算法覆盖。

## 开店服务收益

开店收付款在线下完成。系统只在有效开店事实形成后记录服务收益：

- 县级合伙人初始每单 17600 元；
- 地级、省级、大区初始为 0；
- 字段和版本快照已经保留，可由总部后续配置；
- 取消有效开店事实时，仅对仍待处理的收益生成反向记录。

## 平台加权分红

平台董事可同时拥有两类独立配置：

- 采购链路直接比例，可设为 0；
- 平台总采购业绩加权分红池，初始为 1%。

月度批次按有效采购业绩生成分红池，再按平台董事的有效开店业绩权重分配。批次、受益人和金额都有唯一约束；生成任务重复或并发执行时返回已有批次。

## 页面

- 店长工作台：采购目录、采购单、在途、收货和库存。
- 合伙人工作台：采购收益、退款冲正、开店服务收益、平台分红。
- 总部后台“招商合伙人与开店”：规则、采购收益、开店收益和平台分红批次。

## 数据边界

- 不创建 CRMEB `store_order`。
- 不修改 CRMEB 商品、SKU、商城库存、销量或商城支付。
- 不把采购收益写入商城余额、积分、佣金、提现或旧分销。
- 不允许店员或合伙人创建门店采购单。
- 不把客户门店归属、会员一级推荐和合伙人层级合并为同一关系。
- 既有供应链没有采购售后入口；本轮只提供可复用的幂等反冲服务，不声明采购售后已经上线。
