# 御方通和原生采购订单 V1 运行验证

## 目标

本轮把门店采购从独立采购单运行态收口到 CRMEB 原生购物车、订单、支付、物流、退款和订单后台。旧 YFTH 采购表仅保留历史只读事实，不再承接新采购写入。

## 数据与业务边界

- `yfth_supply_catalog` 继续表示总部采购目录。
- `yfth_supply_catalog_sku` 保存独立采购 SKU 价格，不复用零售价。
- 采购目录可以引用普通商城商品，也可以引用仅供采购、前台隐藏的 CRMEB 商品。
- `yfth_native_procurement_order` 为 CRMEB `store_order` 增加采购来源、门店和幂等快照。
- 采购订单不生成普通商城 C1/B1 佣金，只生成县级、地级、省级、大区和平台五级采购收益。
- 普通商城订单不生成采购收益。
- 旧采购单、审核、发货、收货、入库和预警写入口已停用，历史数据不删除。

## 实际验证环境

- PHP 7.4.33。
- MySQL Community 8.0.46，隔离数据库，表前缀 `eb_`。
- Admin Vue production build。
- H5 / mp-weixin：HBuilderX 5.14 `uniapp-cli` 与 Node.js 18.20.8。

未连接生产数据库，未执行真实支付、短信、微信分账或微信小程序上传。

## 验证结果

### PHP 与 Contract

- 本轮新增和修改 PHP 文件语法全部通过。
- 供应链 contract：45 项通过。
- 采购合伙人收益 contract：31 项通过。
- 自动佣金、Stage 3 商城消费收益、多角色壳层和请求降级检查通过。

### Migration

隔离 MySQL 8.0.46 完成：

- up：采购订单 sidecar、采购 SKU 价格表、字段、唯一索引和菜单正确创建。
- down：本轮结构和菜单正确移除。
- rerun：结构和权限恢复。
- 重复 sidecar 和重复 catalog SKU 被唯一约束拒绝。
- 已存在原生采购业务事实时，回退会拒绝破坏性删除。

### 原生采购流程

- 店长采购目录返回独立采购价格。
- 本地采购购物车生成 CRMEB 原生订单确认入口。
- 原生订单创建时冻结采购来源、门店、SKU 和价格事实。
- 订单后台可识别采购订单及所属门店。
- 采购订单沿用 CRMEB 支付、发货、收货、退款和售后基础能力。
- 旧采购 create/audit/ship/receipt/alert 写入口全部明确停用。
- 六张旧采购历史表在原生采购真实流程前后保持不变。

### 收益隔离

- 采购收益按县/地/省/大区/平台 `20%/10%/5%/3%/1%` 测试规则逐级计算。
- 同一订单重复事件不重复生成收益。
- 采购退款按实际采购收益快照反冲，重复退款幂等。
- 普通商城 C1/B1 listener 跳过采购订单。
- 采购 listener 只处理原生采购订单。
- 县级开店服务收益 `17600.00` 和平台 1% 加权池的既有边界未改变。
