# 招商合伙人、门店归属与统一奖励闭环整改 V1

## 1. 正式口径

- 正式身份统一为“招商合伙人”，职级为县级、地市级、省级、区域董事、平台董事。新开店不再新增 `franchisee` 权限。
- 合伙人与店铺的所有权归属独立于店长/店员权限。正式开店必建合伙人资格和店铺归属，店长权限可选。
- 奖励只认直属第一层关系，不沿合伙人职级链递归发放，不写 CRMEB 余额、积分或佣金账户。
- 直属合伙人的第 1/2/3 家有效门店，按财务确认加盟费的 20%/30%/50%获得商品额度；第 4 家起保留不可奖励记录，金额为 0。
- 永久会员的有效、冻结、失效、退款状态继续由永久会员现状表和事件表管理，历史有效 5980 会员保持兼容。
- C 端永久首店归属继续使用总部权威归属表；商城支付、套餐激活、退款/关闭和开店只通过统一奖励事件入口写新奖励事实。

## 2. 数据模型

正式迁移：`20260720100000_create_yfth_partner_reward_unification_v1.php`。

- `yfth_partner_store_binding`：当前合伙人—店铺所有权。
- `yfth_partner_store_binding_event`：绑定、终止等追加式事件。
- `yfth_reward_event`：统一奖励 outbox、处理状态、退避重试和结果引用。
- `yfth_reward_adjustment_ledger`：确认、结算、取消、部分退款和冲正流水。
- `yfth_partner_opening_quota_award`：直属开店序号、加盟费快照、比例和商品额度。
- `yfth_product_quota_reservation`：采购单商品额度预占、核销、释放、退款和退款冲正。
- `yfth_partner_migration_issue`：历史加盟商确定性迁移异常清单。
- `yfth_partner_profile` 新增资格状态和有效期字段。

迁移会把可确定的历史有效合伙人主店回填为归属；同一店铺出现多个所有者时不猜测，写入异常清单供总部处理。

## 3. 统一事件与失败恢复

事件唯一键为 `SHA-256(event_type|source_type|source_id)`。业务事务先写事件，再尝试处理；处理失败保留 `failed`、错误、重试次数和下一次重试时间。后台“奖励事件治理”可查询和批量重试。五分钟以上的遗留 `processing` 事件可重新认领。

正式入口：

- 套餐激活：写 `package_activated`，使用激活事务内冻结的直属关系快照。
- 套餐退款、冻结或关闭：写 `package_invalidated`，未结算候选取消，已结算事实追加负向冲正。
- 商城支付：写 `mall_order_paid`。
- 商城累计部分/全额退款：写 `mall_order_refunded`，按累计退款额计算差额，追加调整或冲正。
- 正式开店：写 `partner_store_opened`。
- 开店取消：写 `partner_opening_cancelled` 并冲正已授商品额度。

## 4. 商品额度采购闭环

门店采购单支持 `quota_amount_cent`。订单总额减商品额度为在线支付金额，提交时预占额度；总部拒绝时释放；确认入库时核销。售后服务提供部分/全额额度退回以及退回冲正，所有动作写 `yfth_product_quota_ledger`，不修改 CRMEB 用户资产。

五级招商合伙人通过有效资格与 `yfth_partner_store_binding` 解析所属门店上下文；即使没有兼任店长，也可在 C 端选择该合伙人身份查看额度并提交门店采购。店长权限仍是独立、可选的门店经营权限。

## 5. 管理后台与 C 端

- 合伙人详情分别展示直属上级/成员、合伙人归属店铺和可选店长权限。
- 新增开店商品额度、奖励事件治理和历史迁移异常页签。
- 采购页可输入商品额度金额，详情展示“商品额度 + 在线支付”构成及额度状态。

本轮只开发、验证、提交和推送分支，不写生产数据库、不部署、不合并主分支。生产迁移只能在备份、维护窗口、审批和发布清单齐备后执行；禁止在生产使用 `migrate:rollback -t 0`。
