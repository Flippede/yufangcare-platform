# 御方通和线下会员与门店二维码 V1

## 身份边界

唯一业务身份层级为：总部 -> 平台董事 -> 大区总监 -> 省级合伙人 -> 地级合伙人 -> 县级合伙人 -> 门店店长/店员 -> 永久会员/普通用户。

系统不再提供“加盟商”经营身份。开店申请通过后，申请人只获得新门店的店长身份；门店继续归属招商来源合伙人。店长和店员属于门店经营角色，授予时自动激活永久会员资格，但不会因此获得合伙人层级。

## 二维码语义

- 合伙人码：仅用于打开加盟申请并记录招商来源。
- 门店获客码：由店长或店员生成，顾客扫码后只绑定权威 B1 门店，不创建会员推荐关系，也不产生 15%/25%/60% 奖励。
- 会员推广码：普通永久会员用于建立唯一 C1 -> C2 一级推荐关系。
- 用户身份码：普通非会员出示给本店店长/店员，用于线下会员开通；成为永久会员后，同一入口按产品文案显示为推广码。

二维码只携带不可预测的短期业务令牌。用户 UID、门店 ID、角色和奖励资格均由服务端重新解析和校验，前端参数不构成授权依据。

## 线下会员开通

### 顾客申请

顾客从现有 9800 套餐页面提交线下会员申请。申请金额、套餐规则和归属门店由服务端权威数据生成。申请进入本店工作台，店长或店员可批准或拒绝。

### 门店扫码

店长或店员通过工作台会员开通入口扫描顾客身份码。服务端确认操作者角色有效、门店正常、顾客权威归属与当前门店一致后，直接激活永久会员。

两条入口调用同一个会员激活服务，具有相同的门店隔离、事务、审计、幂等和奖励规则。

## 奖励规则

若顾客激活前存在有效的 C1 -> C2 推荐关系，系统先冻结规则、门店、C1、C2、金额和有效序号快照，按既有 15%/25%/60% 阶梯生成佣金事实，成功后再关闭推荐关系。若不存在有效 C1，则只激活会员。

店长/店员的门店获客码不会创建 C1 关系。顾客完成普通商城订单时，门店收益继续按权威 B1 归属计算，不把门店经营人员伪装为会员推荐人。

## 开店与迁移

总部批准开店前必须绑定有效 CRMEB 门店。批准后申请人获得店长身份和永久会员，新门店归属原招商来源合伙人。遗留 active `franchisee` 经营角色通过正向迁移转换为店长；同门店已有店长时停用重复遗留角色。旧合伙人档案链接字段归零，历史迁移和审计记录保留。

## 非目标

- 不新增线上支付或第二套套餐订单。
- 不新增多级会员推荐。
- 不把门店获客关系解释为 C1 推荐。
- 不改变 CRMEB 商品、订单、支付和退款主流程。
- 不自动执行真实微信分账、短信或小程序上传。
