# Superpowers 项目规则

- 用户提供的完整 Codex 指令即为已批准的任务规格。除非存在真实阻塞，不得要求用户再次确认已明确的需求。
- 不得擅自改变任务范围、业务规则、分支名称、Git 基线、验收标准或部署边界。
- 用户指定分支时，不得额外创建分支或 worktree；不得自动合并 `main`，不得自动删除功能分支，也不得自动部署正式服务器。
- 简单样式、文案和布局修改不强制完整 TDD；登录、鉴权、门店权限、订单、支付、推荐、奖励和数据库迁移必须执行适当测试。
- Superpowers 内部 review 不能替代独立的 Architecture Auditor；收到审核报告后仅修复本轮范围内的问题，不得借机全项目重构。
- 完成前必须提供真实、可核验的验证证据。
