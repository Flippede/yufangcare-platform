# 御方通和多身份小程序交互 Demo V1

这是一个本地可点击交互 Demo，用于确认御方通和多身份微信小程序的信息架构与关键流程。

## 如何打开

1. 双击 `index.html`。
2. 推荐使用最新版 Chrome 或 Edge。
3. Demo 不依赖后端、不依赖网络、不访问真实 API。

## 本地服务方式

如果浏览器策略导致本地文件行为异常，可双击：

```bat
serve.bat
```

默认地址：

```text
http://127.0.0.1:18110/
```

## 如何使用

- 顶部演示控制栏可切换身份、经营主体和门店。
- “重置演示数据”会恢复为顾客、郑州金水店和默认预约状态。
- “返回身份首页”会回到当前身份的第一个底部导航页面。
- 手机外框内底部导航会随身份自动变化。

## 推荐体验路径

1. 顾客预约：`首页 -> 康养项目 -> 选择门店 -> 选择日期和时段 -> 确认预约 -> 预约成功 -> 我的预约 -> 打开动态核销码`。
2. 店长处理预约：`切换店长身份 -> 今日待确认预约 -> 查看预约 -> 确认预约 -> 查看客户 -> 进入核销`。
3. 店员核销：`切换店员身份 -> 核销 -> 输入六位数字码 -> 核销预检 -> 确认核销 -> 核销成功`。
4. 加盟商查看多门店：`切换加盟商身份 -> 全部门店总览 -> 进入郑州金水店 -> 查看客户 -> 查看预约 -> 返回全部门店 -> 进入采购流程`。
5. 导师跟进线索：`切换服务导师 -> 线索 -> 查看详情 -> 添加跟进记录 -> 更新状态 -> 查看活动任务`。

## 演示边界

- 所有数据均为本地演示数据。
- 不包含真实手机号、身份证、商户号、密钥或生产配置。
- 采购、加盟合同、奖励台账、收款设置、个人业绩、库存补货等未完成能力均标记为“规划中”或“演示流程”。
- 不修改正式 uni-app 页面、后端 API、数据库迁移或总部 Web 后台。
